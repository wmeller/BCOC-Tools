BCOC Tools

Below, you will find a set of tools for automating a few of the boring tasks we have to do. Enjoy!


VLAN Subnetting Tool
Download VLAN Subnetting Tool xls

I generally have trouble remembering exactly how many hosts are allowed with each subnet size. After a few times of going back and forth with that same VLSM table and my requirements, I put together this spreadsheet to do it for me. Drop your user/host requirements into the given table and it will tell you what size subnet you need for that VLAN in CIDR notation. It will also tell you the total number of each subnet size you need to make it more compatible with the auto subnetting tool. You could also paste in unit NIPR/SIPR host requirements to get the WAN blocks you need to allocate to subordinates.

Auto Subnetting Tool
Go To Auto Subnetting Web Tool
The second addition to this website! Give it your starting block of IP's and the size of your block, then tell it what subnets you need. It will generate a table of IP addresses for you with size, network and broadcast addresses, and your remaining IP addresses in reserve.

SLD Matrix to Tasking Statements Generator
Download SLD to Tasking Matrix xls

This spreadsheet lets you load in your SLD Matrix and it will generate the tasks for you for both MCR links and network links. I wrote this tool for the MCR and networking TDG's, so it may require some updates for future assignments. It should do at least 90% of the work though. At this point, network link speeds are determined by looking at the physical link for that network. The matrix doesn't know anything about multiple links between the same site.

Equipment Matrix to Tasking Statements Generator
Download Equipment to Tasking Matrix xls

The Equipment Matrix to Tasking Statements tool will generate tasking statements from your equipment allocation matrix (duh). There are two sheets. Use the first as a pre-built MCR equipment table, and just fill out numbers. If you want to generate tasks for any old equipment, flip to the second tab and fill out the unit names, equipment type and quantities. You can probably just paste the entire table from the template in, as long as you didn't mess with the template at all.

Demonstrations


Balanced Transmission Line Demonstration
Download Balanced Tx Line Demo xls

Download the spreadsheet and follow the instructions. The purpose of the demo is to visualize what it means to signal balance a transmission line - this is how we reduce noise in very sensitive instruments or when our transmission lines are long and likely to be noisy (like when remoting an antenna hill).

Signal Modulation Demonstration
Download Signal Modulation Demo

This excel file has demonstration for all of the common kinds of modulation used by the Marine Corps. Play around with the data and watch the signals shift to represent the unencrypted RF signal we generate.

Phase Shift Keying Demonstration - Animated
Download Animated PSK Mod Demo

This demonstration generates an animation of Phase Shift Keying Modulation. Download the file, and enable the content. Type the message you want to see into the light blue message box, then click the "Send Signal" button to see the signal go out. (NOTE: This will only work on a machine where macro's are allowed. This is not likely to work on your USMC computer.)

Superimposed Sine Waves Demonstration
Download Sine Wave Superposition Demo

The Sine Wave superposition demo visualizes the fundamental geometry of radio waves - sin waves, and how they can be added together. This is the mathematical principle used for adding information to radio waves, detangling multiple RF signals impinging on a single antenna, and handing data off from one physical device to another. Play around with the characteristics of the two pre-defined waves to see how they superimpose on each other to create a new third wave. (NOTE: If the two signals you choose are 180 deg out of sync, mathematically, you should see a resulting signal of amplitude 0. You won't. Because of how Excel calculates sine values, you will see a small residual value usually of the order of 1E-14, which are basically rounding error.)
